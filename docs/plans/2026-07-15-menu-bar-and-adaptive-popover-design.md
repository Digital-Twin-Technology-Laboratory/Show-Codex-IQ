# Menu bar compression and adaptive popover design

The menu-bar item will size itself from its rendered SwiftUI content instead of adding fixed allowances to a 94-point base width. The two ranking rows use a compact grid with no expanding spacer, one-point outer padding, and a smaller gap between the optional metric symbol and the text. A persisted `showsMenuBarIcon` preference defaults to enabled so upgrades preserve the current appearance. Hiding it removes the complete leading symbol allocation. The rendered content reports its ideal width back to `StatusItemController` whenever rankings or presentation preferences change, allowing the `NSStatusItem` to shrink and grow with the actual text.

Custom model aliases are stored by stable benchmark ID. The menu bar first uses a non-empty alias and otherwise falls back to the existing automatic compact name. Alias editing lives on a dedicated page reached from the Menu Bar section of General Settings. It lists every model in the current snapshot with the full label and an initially empty text field; whitespace-only input removes the saved alias. New models appear automatically when data refreshes. The popover, charts, tooltips, and accessibility descriptions continue to use their existing full or automatically formatted labels and never use the custom aliases.

The popover keeps its fixed 430-point width but replaces the 680/460-point height presets with measured content height. The dashboard's main content and footer report their rendered heights. The popover adopts that ideal height while it fits, so disabling the trend chart, changing error banners, or expanding a ranking removes unused vertical space. Its maximum is the smaller of a 760-point design cap and the active screen's visible height minus a safety margin. The existing scroll view receives only the remaining viewport height; scrolling and indicators become necessary only when measured content exceeds the cap. Empty and loading states retain a small minimum height so they remain usable.

Verification covers default values and persistence for the icon preference and alias dictionary, alias trimming/removal and fallback naming, the complete Swift package tests, an application build when a full Xcode toolchain is available, and visual inspection of the compact menu-bar item and both capped and uncapped popover states.
